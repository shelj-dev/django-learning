Admin Panel — admin.py & Superuser
Django Admin • admin.py

Django provides a powerful built-in admin interface that allows you to manage application data without writing custom dashboards. The admin panel is configured using the admin.py file. 

admin.py

from django.contrib import admin
from .models import Book

# Register model to make it visible in the admin panel
admin.site.register(Book)

Once a model is registered, it becomes accessible through the Django admin interface, where you can add, edit, and delete records.

Creating a Superuser

Terminal

python manage.py createsuperuser

This command creates a superuser, which is an administrator account with full access to the Django project. You will be prompted to enter:

  • Username
  • Email address
  • Password

What is a Superuser?

A superuser is a special Django user who has all permissions enabled by default. Superusers can:

  • Access the admin panel (/admin/)
  • Add, edit, and delete any model data
  • Create and manage other users
  • Assign permissions and groups
  • Bypass object-level permission restrictions

The admin panel can be accessed by visiting: http://127.0.0.1:8000/admin/ and logging in with the superuser credentials.

Tip: Only trusted users should be granted superuser access, as it provides complete control over the application.

Models — Field Types
Django ORM • models.py

Django models define the structure of your database tables. Each model class represents a table, and each field represents a column. Django provides many built-in field types to handle different kinds of data. 

models.py

from django.db import models

class Book(models.Model):
    # Text-based fields
    title = models.CharField(max_length=200)
    description = models.TextField()

    # Numeric fields
    price = models.DecimalField(max_digits=6, decimal_places=2)
    pages = models.IntegerField()

    # Date and time fields
    published_date = models.DateField()
    created_at = models.DateTimeField(auto_now_add=True)

    # Boolean field
    is_published = models.BooleanField(default=True)

    # File and image fields
    cover = models.ImageField(upload_to='covers/', blank=True, null=True)

    # Relationship fields
    author = models.ForeignKey(
        'Author',
        on_delete=models.CASCADE,
        related_name='books'
    )

    def __str__(self):
        return self.title

Common Django Model Field Types:

  • CharField – short text, requires max_length
  • TextField – long text
  • IntegerField – whole numbers
  • DecimalField – precise decimal values (money)
  • DateField – dates only
  • DateTimeField – date and time
  • BooleanField – true / false
  • ImageField – images (requires Pillow)
  • ForeignKey – many-to-one relationship
  • ManyToManyField – many-to-many relationship
  • OneToOneField – one-to-one relationship

After creating or modifying models, always run: python manage.py makemigrations and python manage.py migrate.

Models — Field Options
Django ORM • models.py

Django model fields support many options that control database behavior, validation, and form handling. These options are passed as arguments when defining a field. 

models.py

from django.db import models

class Book(models.Model):
    title = models.CharField(
        max_length=200,
        null=False,
        blank=False
    )

    summary = models.TextField(
        blank=True
    )

    price = models.DecimalField(
        max_digits=6,
        decimal_places=2,
        default=0.00
    )

    STATUS_CHOICES = [
        ('draft', 'Draft'),
        ('published', 'Published'),
    ]

    status = models.CharField(
        max_length=10,
        choices=STATUS_CHOICES,
        default='draft'
    )

    published_date = models.DateField(
        null=True,
        blank=True
    )

Common Model Field Options:

  • null – If True, stores NULL in the database
  • blank – If True, field is optional in forms
  • default – Sets a default value for the field
  • choices – Limits values to predefined options
  • unique – Ensures all values are unique
  • db_index – Adds a database index for faster queries
  • help_text – Displays guidance text in forms and admin

Important note: Use blank=True for form validation and null=True for database storage — they serve different purposes.

After changing field options, remember to run migrations to apply changes to the database schema.
