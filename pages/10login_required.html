Django Login Required

User authentication, login protection, decorators, redirect flow

Django provides the login_required decorator to restrict access to views. Only authenticated users can access protected pages.

Import Required Decorator

Import the decorator from Djangoâ€™s authentication module. 

views.py
from django.contrib.auth.decorators import login_required

Protecting a View

Apply the decorator to any view that requires user login. 

views.py
@login_required(login_url="/myapp/login/")
def home(request):
    return render(request, "home.html")

If a user is not logged in, Django automatically redirects them to the login page.

Login Page URL

The login URL specified in the decorator: 


/myapp/login/

Users must enter valid credentials to continue.

After Successful Login

Once authenticated, the user is redirected to the protected home page.

Access Control Flow

The authentication flow works as follows:

  • User tries to access protected page
  • Django checks authentication
  • Redirects to login page if not logged in
  • Allows access after successful login

This ensures that sensitive pages are accessible only to authorized users.
